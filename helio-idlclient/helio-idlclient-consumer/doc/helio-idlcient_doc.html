Helio IDL-Client Doc:

If more than one command is provided for a method, they are equal. For all following commands o is the heliovo object created by command 1) or 2), s is the heliovo_service object created by command 8) or 9) and c is the heliovo_catalog object created by command x) y).

Heliovo object

Object instance creation

An instance of an heliovo must be created first. Here is the idl command to do this.

1) IDL> o=obj_new('heliovo')
2) IDL> o->heliovo()

Help

This command can be used to print a small help. This help contains the help for all heliovo objects

3) IDL> o->help

Print methods

All available helio services can be printed with this command.

4) IDL> o->print, /service
5) IDL> o->print_service

Find methods

This command proveds a search function to find a specific helio service. RegExp is used for the search string.

6) IDL> o->find, service='RegExp'
7) IDL> o->find_service, 'RegExp'

Get methods

Create and return a helio_service object for the specified helio service. Service can be specified by id or name. Service id is shown by the commands 4-7.

8) IDL> s = o->get(service='SERVICENAME')
9) IDL> s = o->get_service('SERVICENAME')

Return the internal datastructure of the heliovo object (heliovo_framework object)

10) IDL> data = o->get(/data)
11) IDL> data = o->get_data()

heliovo_service object

Help

This command can be used to print a small help. This help contains only the help for this object

12) IDL> s->help

Print methods

All available helio catalogs of this helio service can be printed with this command.

13) IDL> s->print, /service
14) IDL> s->print_catalog

Find methods

This command proveds a search function to find a specific helio catalog of this helio service. RegExp is used for the search string.

15) IDL> s->find, catalog='RegExp'
16) IDL> s->find_catalog, 'RegExp'

Get methods

Create and return a helio_catalog object for the specified helio catalog. Catalog can be specified by id or name. Catalog id is shown by the commands 13-16.

17) IDL> s = o->get(service='SERVICENAME')
18) IDL> s = o->get_service('SERVICENAME')

Return the internal datastructure of the heliovo_service object (heliovo_framework object)

19) IDL> data = o->get(/data)
20) IDL> data = o->get_data()

heliovo_catalog object

Help

This command can be used to print a small help. This help contains only the help for this object

21) IDL> c->help

Set method

One or more time_intervals must be set. With this command the time_interval can be set. All date formats accepted by anytim() can be used.

To set more than one time_interval a 1-dimensional array (command 23) with the syntax [start1,end2,start1,end2,...] or a 2-dimensional array (command 24) with the syntax [[start1,end1],[start2,end2],...] can be used.

22) IDL> c->set, time_interval=['1-may-2004','2-may-2004']
23) IDL> c->set, time_interval=['1-may-2004','2-may-2004', '1-may-2005','2-may-2005']
24) IDL> c->set, time_interval=[['1-may-2004','2-may-2004'],['1-may-2005','2-may-2005']]

Get methods

Start a helio query and directly parse the data to a struct. This function streams the votable from the url given by the result of the query. To save the votable local and parse it step by step skip this command and follow the instruction below (26-35)

25) IDL> data = c->get(/struct)

Start a helio query and return the result as a helio_framework object.

26) IDL> result = c->get(/query)
27) IDL> result = c->get_query()

Return the internal datastructure of the heliovo_catalog object (heliovo_framework object)

28) IDL> data = c->get(/data)
29) IDL> data = c->get_data()

helio_framework

The heliovo_framework object is used for the communication between idl and the java service.

Show method

The show method is used to see all available attributes inside the object.

30) IDL> print, object->show()
31) IDL> object->show

Get method

With this method all data inside the heliovo_framework object can be accessed. Use /attr to get the attribute attr from the object

32) IDL> x = object->get(/attr)

votable parser

Object instance creation

An instance of the parser must be created first. Here is the idl command to do this.

33) IDL> parser = obj_new('votable2struct')

Parse votable

With this command a votable file can be parsed. The return value is a struct with all the meta information from the votable.

34) IDL> data = parser->getdata('FILENAME')

Parse data only

With this method only the data inside the votable is parsed. The Meta information ist ignored. The return value is simple data array.

35) IDL> data = parser->gettable('FILENAME')
